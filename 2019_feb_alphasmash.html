AlphaSmash: Building a Bot for Super Smash Bros. Ultimate Using Deep Learning

In February 2019, AlphaSmash was developed as part of the HackIllinois 2019 hackathon. Inspired by the AI systems AlphaGo and AlphaStar, we set out to create a bot that could challenge and potentially beat friends in Super Smash Bros. Ultimate on the Nintendo Switch. The project combined deep Q-learning and computer vision to train a reinforcement learning agent capable of playing the game autonomously.

Key Contributions

  • Developed a bot using Deep Q-learning and computer vision to continuously learn and improve in Super Smash Bros. Ultimate.
  • Designed a custom setup using Arduino, relays, a webcam, and controllers to interact with the Nintendo Switch in real time.
  • Integrated OCR to interpret in-game health percentages, allowing the bot to evaluate its performance and adjust its strategies.

Key Project: AlphaSmash - Learning to Play Super Smash Bros.

The goal of AlphaSmash was simple yet ambitious: train an AI to play Super Smash Bros. Ultimate autonomously and effectively. The inspiration came from advanced AI systems like AlphaGo and AlphaStar, and we applied similar principles using a mix of hardware and machine learning techniques.

Setup

The system consisted of a compute machine, Arduino Zero, Nintendo Switch, webcam, controllers, and mechanical relays. To reduce costs and latency, we avoided using a capture card and instead relied on a webcam to record the game screen. The webcam feed was then processed by a CNN + DQN (Deep Q-Network) model, which identified the current game state.

Action Generation & Control

The CNN processed visual inputs, while the DQN generated an action vector based on the game state. These actions, which corresponded to controller inputs, were physically sent to the Switch using mechanical relays controlled by the Arduino. This bypassed the need to directly hack into the Switch, while still allowing our system to interact with the game.

Feedback Loop

In-game health percentages, captured through OCR, provided the reinforcement agent with feedback on performance, allowing it to update its Q-values and improve over time. Transfer learning was also used to speed up the training process by applying a pre-trained CNN model to interpret visual data more efficiently. The ultimate goal was to train the AI solely against in-game bots, creating an automated training loop for continuous improvement.

Challenges

Though the system worked, several challenges arose, particularly balancing hardware and software interactions. The use of mechanical relays to control the Switch introduced some response time limitations, while using a webcam instead of direct video input added noise to the visual data. Additionally, the short timeframe of the hackathon meant we couldn’t fully train the AI model, but early results showed promise, as the bot exhibited basic patterns like moving, attacking, and responding to in-game events.

Despite the bot occasionally getting stuck in repetitive loops (such as moving to the left platform, attacking, then moving to the right and falling off), these patterns were proof that the system was learning and reacting to the game environment, albeit in an incomplete form.

Technologies Used

  • Programming: Python, Pytorch
  • Machine Learning: CNNs, Deep Q-Learning, Transfer Learning
  • Computer Vision: OpenCV, OCR
  • Hardware: Arduino Zero, Mechanical Relays, Nintendo Switch
  • Other: GitHub, Reinforcement Learning

While we didn’t reach a fully trained AI in the time available, AlphaSmash was an exciting project that explored the potential of combining hardware and AI for gaming. You can view the progress and code on the GitHub repository.
